Tag HTML <br>

Basta inserir a tag <br> no local exato onde a linha deve ser quebrada. Por ser um elemento vazio (void element) ele não possui tag de fechamento;

Exemplo simples

O elemento <br> é usado para forçar uma quebra de linha dentro de um bloco de texto, como um parágrafo ou um cabeçalho. É útil em situações onde a divisão de linhas é parte do conteúdo, como em um endereço.

<p>Google Brasil Ltda.<br>
Av. Brigadeiro Faria Lima, 3477<br>
São Paulo, SP</p>

Definição e uso

O elemento <br> (Line Break) insere uma única quebra de linha. Ele é um elemento vazio, o que significa que não possui uma tag de fechamento (não se usa </br>).

Seu uso é estritamente para casos onde uma quebra de linha é semanticamente parte do conteúdo. Ele não deve ser usado para criar espaçamento visual entre elementos.

  • Poesia: Para separar os versos de um poema.
  • Endereços: Para formatar as linhas de um endereço postal.
  • Música: Para separar as linhas de uma letra de música.

Para criar separação entre blocos de conteúdo, como parágrafos, deve-se usar a propriedade CSS margin.

Atributos

O elemento <br> não possui atributos específicos e apenas aceita os atributos globais do HTML.

Atributo Valor Descrição
id Texto Define um identificador único para o elemento.
class Texto Permite associar o elemento a uma classe CSS, embora estilizar <br> seja incomum.

Observação: O atributo clear, que era usado para controlar o fluxo de texto ao redor de elementos flutuantes, foi depreciado em HTML5. A funcionalidade equivalente deve ser implementada com CSS.

Boas práticas e observações

  • NÃO use <br> para criar margem: O erro mais comum é usar múltiplas tags <br><br> para criar espaço vertical. Esta é uma prática ruim que prejudica a acessibilidade e a manutenção. Use CSS margin para controlar o espaçamento.
  • Elemento Vazio (Void Element): Em HTML5, a sintaxe correta é simplesmente <br>. A sintaxe de auto-fechamento (<br />) é herança do XHTML e, embora seja tolerada pelos navegadores, não é necessária.
  • Acessibilidade: O uso excessivo de quebras de linha pode confundir leitores de tela. É preferível estruturar o conteúdo com elementos de bloco apropriados (<p>, <div>, <ul>, etc.).
  • Diferença entre <p> e <br>: <p> define um parágrafo (um bloco semântico de texto), enquanto <br> apenas cria uma quebra de linha visual dentro de um bloco existente.

Exemplos de uso

1) Formatando um poema

<p>No meio do caminho tinha uma pedra<br>
tinha uma pedra no meio do caminho<br>
tinha uma pedra<br>
no meio do caminho tinha uma pedra.</p>

2) Uso incorreto (a ser evitado)

O exemplo abaixo usa <br> para criar espaço entre dois parágrafos. Esta é a forma errada de fazer.

<!-- Forma Incorreta -->
<p>Primeiro parágrafo de texto.</p>
<br>
<br>
<p>Segundo parágrafo de texto.</p>

3) A forma correta de criar espaçamento (com CSS)

A maneira correta de controlar o espaçamento é usando CSS, por exemplo, definindo uma margem inferior para os parágrafos.

<!-- CSS -->
p {
  margin-bottom: 24px;
}

<!-- HTML (Forma Correta) -->
<p>Primeiro parágrafo de texto.</p>
<p>Segundo parágrafo de texto.</p>
